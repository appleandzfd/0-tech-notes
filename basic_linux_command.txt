在执行命令的时候，如果路径中有空格等，会导致执行命令失败。

e.g svn checkout
svn checkout http://172.31.88.114:8443/svn/Android/GUI/T882 2.0
执行上面命令会出错，原因是路径中含有空格。

为了执行正确，需要加上引号来显示完整路径。
svn checkout 'http://172.31.88.114:8443/svn/Android/GUI/T882 2.0'
这样就能够正常的checkout并且更新了。

Basic Linux command

1. linux 下如何打开pdf文档？
   使用自带的pdf浏览器打开：evince xx.pdf (可以以后台运行&)
   使用acrobat来打开：acroread xx.pdf(同上，否则此终端一直运行不停止)

2. 如何使用上一条命令中的参数或者选项?
   使用 Esc + . 的组合能够在命令行下面直接使用上一条命令的参数和选项.

3. 历史搜索功能?
   如何搜索某条历史记录,这样就不用一直按方向键来选择某个命令.
   Ctrl + r 能够调出搜索命令,然后输入相关的单词,就能够按照最近到最远的顺序列出相应的命令
   找到之后,直接回车就可以执行对应的命令.

4. 后台运行某个程序.
   命令行中启动某个程序需要在控制一直运行的情况下,要是想不依赖终端而后台执行,则可以加上&符号.
   如果已经启动了,不想退出来再次重新输入.可以先Ctrl + Z 中断执行, 然后bg, 说明以后台运行这个
   暂停的jobs. 注意Ctrl + Z之后也可以fg, 这样就会到前台重新执行(Ctrl+z会中断,暂停当前执行.)

5. 如何递归删除某个目录下面的.svn目录及下面对应的文件?
   find -type d -name .svn|xargs rm -rf i # 注意一定要是目录类型,否则的话可能会导致删除所有的文件.

6. 自动化配置脚本的问题.为何执行./后环境变量配置仍然不成功?
   场景: 在某个脚本中,执行source xx.sh, 然后利用choosecomobo命令选择一些选项后, 发现不起作用.
   原因: 因为只对当前脚本的bash有效,所以以./允许的方式,后面会重新设置为父shell的环境变量,执行完
         那个脚本,基本上就没有作用了.
   解决办法: 使用source命令,让修改同时应用到父shell中.
